### Sprint 25 — Listes d'épicerie dans le frigo (partagées, cochage à deux, push anti-spam) ✅
**Feature dirigée par l'auteur** (définie en session ; hors FR-1→17, consignée). Prolonge l'espace du frigo (Sprints 20-24) vers un nouvel usage : des **listes d'épicerie** où **les deux membres** créent une liste, ajoutent/retirent des articles et **cochent/décochent** ce qui est acheté. **Réconciliation d'abord** (loi anti-hallucination) : tout le code épicerie était **à créer** (aucune table/type/action `epicerie` n'existait — vérifié) ; le helper RLS `is_household_member` (`supabase/migrations/20260611192623_rls_policies.sql:14`) et l'infra push (`push_subscriptions`, VAPID, Resend, gabarit `notify-fridge`) **existaient** et ont été réutilisés (aucun nouveau secret/env). **Décisions produit (auteur, en session)** : (1) **gestion à deux** (ajouter/retirer/cocher ouverts aux deux membres) ; (2) **placement** = section dédiée « Épicerie » dans `/frigo`, séparée du tableau de notes ; (3) **push au cochage anti-spam** (le 1ᵉʳ cochage d'une liste notifie le conjoint, puis silence pendant une fenêtre de cooldown). **Schéma** : 2 tables (migration `20260628120000_sprint25_grocery_lists.sql`) — `grocery_lists` (titre, `last_check_notified_at` = débounce **interne**, jamais sélectionné/forgeable côté client) et `grocery_items` (`is_checked`/`checked_by`/`checked_at`, `household_id` dénormalisé pour RLS+Realtime, CHECK de cohérence). **RLS partagée** : SELECT/INSERT/DELETE = membre du foyer ; **aucun grant/policy UPDATE** → l'état de coche ne bascule **que** par le RPC `cocher_element_epicerie` (SECURITY DEFINER, `search_path=''`, calque `marquer_note_frigo_lue`) qui vérifie `is_household_member` (**membre révoqué refusé**), est idempotent, et tient le débounce ; INSERT durci (coche non forgeable) ; trigger `enforce_grocery_item_household` (anti-injection cross-foyer, INSERT+UPDATE). **Realtime** : les 2 tables publiées (garde d'existence comme `fridge_notes`). **Couche pure** : `lib/epicerie/{types,db-rows}.ts` — `GROCERY_LIST_COLUMNS`/`GROCERY_ITEM_COLUMNS` (source unique), frontière Zod `parseGrocery*Rows`, helpers `grouperListes`/`compteRestant`. **Actions** `app/epicerie/actions.ts` (`getUser()` + RLS + `revalidatePath`) ; push best-effort via `notify-grocery-client.ts` → Edge `notify-grocery` (calque `notify-fridge` ; destinataire = l'autre membre ≠ acteur). **R7** : payload `grocery-payload.ts` + copie Deno (parité testée) ne portent **que** le type d'événement, jamais un libellé d'article. **UI** `components/epicerie/listes-epicerie.tsx` (cases ☑/☐ ≥ 44 px icône+texte, libellé barré + « coché par X », badge « X à acheter », mise à jour optimiste, Realtime), intégrée à `app/frigo/page.tsx` ; i18n `fr.epicerie`. **Preuve d'acceptation observable** (Chromium réel sur `/demo/epicerie`, **0 erreur JS applicative** — seul un 404 favicon bénin, confirmé par curl) : liste seedée « 2 à acheter », article « Pain » coché « par Conjoint(e) » ; cocher « Lait » → « 1 à acheter », décocher → « 2 à acheter » ; création d'une 2ᵉ liste + ajout d'article. **Revue indépendante** `/code-review` high (8 angles, contexte frais) : **0 bug de correction / 0 faille RLS / 0 violation CLAUDE.md** confirmés (candidats réfutés : `upsertItem` n'affecte pas l'ordre car `grouperListes` re-trie ; `revalidatePath("/")` inutile, pas d'indicateur épicerie sur l'accueil) ; passe `/simplify` : 1 import mort `useRef` retiré (duplications `functionsBaseUrl`/garde d'auth gardées — parité voulue avec le frigo). Gates mesurés : vitest **336** (+36 : 18 isolation RLS épicerie **contre un vrai Postgres** via `scripts/local-db.sh` — isolation foyer / les deux cochent / révoqué refusé / coche non forgeable / UPDATE direct refusé / débounce / cascade ; +8 helpers purs ; +6 frontière Zod ; +3 payload ; +1 parité), tsc 0, biome 0, build OK. **Tâches opérationnelles** (hors session) : appliquer la migration Sprint 25 sur Cloud + régénérer les types ; **déployer l'Edge `notify-grocery`** (+ vérifier `SUPABASE_SERVICE_ROLE`/URL functions). Branche : `claude/prompt-executer-sprint-494ccj`.
