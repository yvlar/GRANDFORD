### Sprint 17 — Jour de paye (worker-private, déterministe) ✅
**Feature dirigée par l'auteur, hors FR-1→17** (consignée : ce n'est pas dans `docs/analyse/` et `CLAUDE.md` exclut « tracker de paie/OT$/banques »). Cadrage retenu : **marqueur de DATE de paye** (quand la paye tombe), jamais un suivi de montants — visible du **seul travailleur**. **Modèle déterministe** (comme le moteur Pitman, « jamais un CRUD de tous les jours ») : on persiste **une config** (ancre + fréquence), les jours de paye se calculent à la volée. **Moteur pur** `lib/schedule/payday.ts` : `estJourDePaye(date, reglage)` réutilise `civilToDays`/`parseCivilDate`/`mathMod` — `mathMod` gère les dates AVANT l'ancre ; fréquences `hebdomadaire` (7 j) / `aux_2_semaines` (14 j) ; `frequencePayeSchema` Zod. **BD** : table `payday_settings` (PK `household_id,profile_id`) — migration `20260624120000_sprint17_payday_settings.sql`. **RLS étanche (R7)** calquée sur `exception_private_owner_only` : `using (profile_id = auth.uid())` (la conjointe, même membre, lit **0 ligne**) ; `with check (profile_id = auth.uid() AND is_household_member(household_id))` — durcissement issu de la revue : empêche d'écrire une paye vers un foyer dont on n'est pas membre (intégrité multi-tenant). **Câblage worker-only** : `app/page.tsx` ne fetch `payday_settings` que dans la branche `role === "worker"` → `reglagePaye` passé à `VueCoupDoeil` (null pour la conjointe, absent par construction). **UI** : marqueur 💰 (`MarqueurPaie`, aria-label — jamais la couleur seule, NFR-12) sur les cases du mois + badge « Paye aujourd'hui » dans le coup d'œil ; formulaire de réglage (ancre + fréquence) dans `/foyer`, server action `definirReglagePaye` (Zod + `getUser()` + upsert sous RLS). **Types** : entrée `payday_settings` ajoutée à `lib/database.types.ts` (insertion fidèle au générateur — régénération en masse impossible : le Postgres local n'a pas pg_graphql, ce qui aurait supprimé `__InternalSupabase` + `graphql_public` présents en prod ; à régénérer par l'auteur sur Cloud). **Preuve d'acceptation observable** : `/demo/horaire?paye=2026-06-04@aux_2_semaines` — vue **travailleur** = 2 marqueurs 💰 (4 et 18 juin) + badge « Paye aujourd'hui » ; vue **conjointe**, paramètres identiques = **0 marqueur, 0 badge** (non-fuite R7 constatée au rendu). **Revue indépendante** `/code-review` high (4 angles, contexte frais) : 0 bug de correction ; 1 durcissement RLS appliqué (clause membre dans `with check`) + 1 test ajouté ; reste = patterns locaux intentionnels. Gates mesurés : vitest **219** (+19 : 13 moteur paye + 6 isolation RLS, **exécutés contre un vrai Postgres** via `scripts/local-db.sh`), tsc 0, biome 0, build OK. Branche : `claude/prompt-executer-sprint-ci2ohp`.
