### Sprint 29 — Marqueur « temps supplémentaire » sur l'accueil + légende en carte + « aujourd'hui » renforcé ✅
**Feature dirigée par l'auteur** (UI/UX, suite Sprints 26-28 ; périmètre cadré en session). Piste « soin visuel de l'accueil » resserrée sur **trois points** : (1) **renforcer la surbrillance « aujourd'hui »** ; (2) **légende en carte bordée** ; (3) **couleur + icône pour les journées en temps supplémentaire (OT)** dans le calendrier. **Réconciliation d'abord (loi anti-hallucination)** : les points 1-2 sont du pur style, mais le point 3 **ne l'est pas** — l'OT est modélisé comme `effect: "working_extra"` (`lib/schedule/types.ts:8`, CHECK DB `initial_schema.sql:102`), or cette distinction était **perdue** au passage `exceptions → DayStatus` : `DayStatus` ne portait que `kind`+`fromException` ; une journée d'OT s'affichait comme une case `jour`/`nuit` ordinaire avec le **point d'écart générique** (`MarqueurEcart`). **Décision produit (R7) — l'OT montré aux DEUX rôles** : la politique de confidentialité énumère explicitement *« présence, absence, **temps supplémentaire** — jamais le motif dans la vue partagée »* (`lib/i18n/fr.ts:314`) ; `working_extra` est un **effet partageable** (table `exceptions` partagée), seul le **motif** `"ot"` est privé (`exception_private`, worker-owner-only) — distinguer une journée d'OT ne divulgue aucun motif. **Livré (couche horaire pure)** : champ **`overtime: boolean`** ajouté à `DayStatus` (`lib/schedule/types.ts`), calculé en un seul site — `overtime: exc?.effect === "working_extra"` dans `dayStatuses` (`lib/schedule/status.ts`, `exc` hissé pour dédupliquer le `byDate.get`). Le jour de récupération (sommeil) **du lendemain** d'un OF de nuit n'est **pas** lui-même de l'OT (lit l'écart du jour courant). **Livré (UI, `components/horaire/vue-coup-doeil.tsx`)** : constante `AFFICHAGE_SUPPLEMENTAIRE` (source unique couleur+icône+label : ⚡ + « Temps supplémentaire » + `bg-supplementaire`) ; composant local `MarqueurSupplementaire` (badge de coin ⚡, `aria-label`, calque `MarqueurEcart`/`MarqueurPaie`) rendu quand `day.overtime` — **remplace** le point générique (un OT *est* l'écart, un seul marqueur) dans la grille mois ET la bande semaine, + puce OT sur la carte « Aujourd'hui » ; **légende** enveloppée dans une **carte bordée** (`rounded-xl border border-neutral-800 bg-neutral-900 p-3`) avec pastille OT ajoutée (les deux rôles) ; **ring « aujourd'hui »** renforcé `ring-2 ring-white` → `ring-2 ring-white ring-offset-2 ring-offset-neutral-950` (bande semaine + grille mois). **Couleur** `supplementaire` (orange, distinct de l'ambre `quart-jour`) ajoutée à `tailwind.config.ts` ; **i18n** `fr.horaire.supplementaire` (aucune chaîne en dur, NFR-13) ; **param démo** `?ot=YYYY-MM-DD` (`app/demo/horaire/page.tsx`, écart `working_extra`). **Périmètre** : frontend + couche horaire pure — **aucune migration, RLS, secret ni donnée nouvelle** (`overtime` dérivé de `effect` déjà chargé). **Preuve d'acceptation observable** (Chromium réel via Playwright, `next dev` `GRANDFORD_DEMO=1`, **0 erreur JS applicative** — seul un 404 favicon bénin, confirmé par curl) : travailleur `/demo/horaire?ot=2026-06-11&date=2026-06-11` → **2** marqueurs ⚡ orange (bande semaine + case mois) sur le 11, puce OT sur la carte « Aujourd'hui », pastille ⚡ dans la **légende bordée**, anneau renforcé (`.ring-offset-2` ×2) ; conjointe `?role=spouse&ot=…` → **même** marqueur OT visible, **0 fuite de motif** (payload ne porte que `effect`) ; sans `?ot=` → **0** marqueur dans le calendrier (régression zéro) ; capture d'écran vérifiée. **Revue indépendante** (sous-agent dédié, contexte frais, effort high, critères d'acceptation fournis) : **0 bug de correction / R7 confirmé étanche** (`overtime` dérive du seul `effect` partagé, `parseExceptionRows` ne blanchit que `on_date`/`effect`/`shift`, `lib/ical/generate.ts` a son type local → `overtime` requis ne casse aucun consommateur, tsc le confirme) ; 1 finding LOW traité — dérive de citation `fr.ts:313`→`:314` corrigée dans 2 commentaires (l'ajout de la clé i18n avait décalé la ligne) ; passe `/simplify` : **0 changement** (diff jugé bien nivelé — `AFFICHAGE_SUPPLEMENTAIRE` source unique, parallélisme voulu avec les marqueurs frigo/paye). Gates mesurés : vitest **209 passés** (+5 tests OT vs 204 ; 132 isolation RLS ignorés faute de Postgres local — sprint sans schéma/policy, isolation non requise), tsc 0, biome 0.
